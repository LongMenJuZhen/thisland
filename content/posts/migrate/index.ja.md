普通のアーキテクチャ移行は、古いものから新しいものへ進むのが定石なのに、どうして逆方向への移行があるのかって?

どういうことだって、[Astroが7.0にアップグレードされた](https://astro.build/blog/astro-7/)から、Astro 6 から Hugo への移行を決断したって?Hugo は 2013 年からの古い歴史があるし、Astro が登場したのは 2021 年だ。Astro は Hugo より「モダン」なこと、まる 8 年分。

## Astro はコンテンツ駆動として不够

Astro は自らをコンテンツ駆動のフレームワークだと称しているが、私から見ると、一言で言えば、Astro は「みんなと共通のテンプレートを共用して各自が設定をパーソナライズする」のではなく、「自分で完全な静的サイトプロジェクトを維持する」ことを前提にしている。Astro は「コンテンツ駆動の Web プロジェクト」であり、「設定を少し追加すれば配信できるコンテンツ」ではないのだ。

具体的に言うと、Astro は Hugo のようにテーマテンプレートを依存関係として管理しない。Web プロジェクトを初期化する時にすべてのソースコードをローカルにクローンしなければならないし、設定ファイルがどこにあるか、どのファイルなのかは自分(あるいはテーマの作者)が明示的に指定する必要がある。Hugo のように `config\_default` フォルダに指定する、というわけにはいかないのだ。

另一方面、Astro で独自のコンポーネントを定義したい場合、ローカルのテーマコピーを変更する必要がある。開発中は直感的に見えるが、2 ヶ月後に原作者がテーマを更新したらどうする?自分でマージしなければならない!

確かに今は AI があるが、だからといって問題は簡単にならない。コードが衝突したら結局取舍を迫られるし、AI もどちらを残したいかを理解してくれるとは限らない。

### 另一方面、(本当にいるのか?)ある視聴者はこう聞くだろう

最初は「ブログサイトなんか作らない、個人のホームページだけ作る」って言ってなかったか?「ブログなんて Web 1.0 時代の代物はとうに淘汰されるべき」って言ってなかったか?

確かにその通りだ。しかし、Douyin のクリエイターだって脚本を書かないわけではあるまい。バイブモーションや AI 生成動画の発展はますます速くなっており、脚本から動画への自動変換も現実になりつつある。であれば、もう脚本を書いた以上、どこかに掲載するならこのサイト以外どこに出す?中国版 Quora こと知 X にでも出すか?知 X のレコメンドときたら推送してくる内容ときたら、もうちょっと質が落ちないかと。自前でサイトを作って、検索エンジンにクロールさせたほうがマシだ。

## MDX は本当にいいのか

Astro は MDX を使うが、Hugo はショートコードを使う。2025 年に技術選定をしていた頃は、この点が私を Astro 寄りにしていた。しかし今振り返ると、その判断は間違っていた。

- MDX はオープンなファイルフォーマットのように見えるが、実は React と深く結びついている(これは AI が言っていたことで、私も正確な意味はよくわかっていない)

- MDX は Markdown の中に JS スクリプトを追加する。周知の通り、JS は別に良い言語とは言えない。その上、コンテンツとコードの境界を曖昧にするし、MDX 中のコードは互いに依存し合えるため、複雑度が跳ね上がる。肝心のコンテンツに対しては、この複雑度は少々 overkill だ。

  - 一方 Hugo のショートコードは見通しがいい。要素間に依存関係はないし、最悪でも括弧が少し増えるだけだ。また私としても、Markdown の文法を拡張するのはなるべく避けたい。せいぜい、特定の Markdown 要素を別のレンダリングにしたい程度だ。

## コメント欄

另外、誰も気にしていない角落里で、実はコメント欄も Giscus に乗り換えて、GitHub のサーバーを間借りさせてもらっている。

なぜか?Waline のようなサービスはデータベースに依存しているが、1 ヶ月アクセスがないと休止してしまうのだ。

そう、小規模サイトとしては、毎月誰かが見に来てくれる保証がないのは本当に心許ないが、かといってハートビートスクリプトを書くのも気が進まない。それはまるでデータベースプラットフォームを騙しているようで(急にこんな高尚な気持ちになっているのは自分でもわからない)。なので、Giscus を使って GitHub の力を借りてコメント欄を実現することにした。

GitHub はもしかしたら变质するかもしれないが、潰れることはない。

なので、もしコメントしたいなら、面倒でも GitHub アカウントを作ってもらえると助かる。せっかく GitHub に来たついでに、このブログに🌟をつけていってもらえると嬉しい。
